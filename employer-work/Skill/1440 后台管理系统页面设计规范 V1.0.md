
> 适用范围：平台级业务系统、数据管理系统、医疗健康信息系统、政务系统、SaaS 管理后台  
> 设计基准宽度：**1440px**  
> 目标：形成统一的 UI 设计基线，确保前端开发、产品设计、原型设计（Axure / Figma / Vue）一致可落地

# 一、总体布局规范（Layout System）

## 1.1 页面基础尺寸

| 项目       | 规范值        | 说明         |
| -------- | ---------- | ---------- |
| 设计宽度     | **1440px** | 标准后台系统推荐宽度 |
| 最小支持分辨率  | 1280px     | 允许横向滚动     |
| 最大常见分辨率  | 1920px     | 居中显示       |

| 内容区域最大宽度 | 1440px     | 居中容器       |
| 页面最小高度   | 100vh      | 全屏布局       |

# 结论（工程建议）

对于你当前这类：

- 平台级系统
    
- 多模块系统
    
- 数据管理系统
    
- 医疗健康系统
    

**1440 是最稳的设计基线**，优于 1920。

原因：

- 兼容 1366 / 1440 / 1920
    
- 不会浪费空间
    
- 医院 / 学校 / 政务电脑适配率最高
    
- 前端开发成本最低
