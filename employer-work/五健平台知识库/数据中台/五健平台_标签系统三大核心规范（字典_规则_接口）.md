# 五健管理平台
## 标签系统三大核心规范（Tag Dictionary + Rule DSL + API Contract）

---

# 一、标签字典设计规范（Naming + Code + Type）

本规范用于统一标签命名、编码规则及标签类型分类，是标签系统治理的基础标准。

---

## 1.1 标签命名规范（Naming Convention）

### 基本原则

```text
语义清晰
业务一致
可长期稳定
避免歧义
```

---

### 推荐命名结构

```text
业务对象 + 状态 / 特征 / 风险
```

---

### 示例（五健场景）

| 推荐名称 | 不推荐名称 | 原因 |
|----------|------------|------|
体重状态 | BMI标签 | 用户不易理解 |
视力异常 | 视力问题 | 含义模糊 |
心理高风险 | 心理异常 | 风险级别不明确 |
未完成随访 | 随访问题 | 不够具体 |

---

## 1.2 标签编码规范（Tag Code）

标签编码必须满足：

```text
唯一
稳定
可读
可扩展
```

---

### 推荐编码结构

```text
业务域_对象_标签
```

---

### 示例

| 标签名称 | 标签编码 |
|----------|----------|
体重状态 | HEALTH_WEIGHT_STATUS |
视力状态 | HEALTH_VISION_STATUS |
心理风险等级 | HEALTH_MENTAL_RISK_LEVEL |
未完成随访 | FOLLOWUP_NOT_COMPLETED |
高危肥胖 | HEALTH_OBESE_HIGH_RISK |

---

### 编码规则

```text
使用大写字母
使用下划线分隔
避免中文
避免数字开头
```

---

## 1.3 标签类型设计（Tag Type）

标签类型用于区分标签业务属性，必须在系统中统一定义。

---

### 推荐标准类型

| 类型 | 说明 | 示例 |
|------|------|------|
基础标签 | 描述基本信息 | 性别 / 年龄段 |
健康标签 | 健康状态 | 体重状态 / 视力状态 |
风险标签 | 风险等级 | 高危肥胖 |
行为标签 | 行为或状态 | 未完成随访 |
运营标签 | 运营属性 | 高活跃家长 |
设备标签 | 设备绑定状态 | 已绑定体脂秤 |
AI标签 | 模型预测结果 | 近视风险预测 |

---

## 1.4 标签值类型（Value Type）

| 类型 | 说明 | 示例 |
|------|------|------|
ENUM | 枚举值 | 正常 / 异常 |
BOOLEAN | 布尔值 | 是 / 否 |
NUMBER | 数值 | BMI值 |
TEXT | 文本 | 风险说明 |
DATE | 日期 | 随访日期 |

---

## 1.5 标签生命周期字段（标准）

```text
is_active
expire_days
created_at
updated_at
```

# 二、标签规则设计规范（Rule DSL）

本规范用于统一标签生成规则表达方式，确保规则可读、可维护、可执行。

---

## 2.1 规则设计基本原则

```text
可读
可配置
可审计
可扩展
```

---

## 2.2 规则结构（标准模板）

```text
规则编号：RULE_001
规则名称：高危肥胖识别
标签：HEALTH_OBESE_HIGH_RISK

条件：
BMI >= 95
AND 年龄 >= 6

动作：
添加标签

有效期：365 天

优先级：3
```

---

## 2.3 Rule DSL 推荐语法

```text
IF 条件
THEN 标签
ELSE 不处理
```

---

## 示例（五健）

```text
IF 裸眼视力 < 4.6
THEN 标签 = 视力异常
```

---

## 支持逻辑运算符

```text
AND
OR
NOT
```

---

## 支持比较运算符

```text
>
<
>=
<=
=
!=
```

---

## 支持时间条件

```text
最近 30 天
超过 7 天
连续 3 次
```

---

## 示例（时间规则）

```text
IF 最近 30 天 未完成随访
THEN 标签 = 随访超期
```

---

## 2.4 规则类型分类

| 类型 | 说明 | 示例 |
|------|------|------|
计算规则 | 根据数值计算 | BMI计算 |
状态规则 | 根据状态判断 | 未复诊 |
时间规则 | 根据时间判断 | 超期随访 |
事件规则 | 根据行为触发 | 已查看报告 |
模型规则 | AI预测 | 近视风险 |

---

## 2.5 规则审计字段（必须）

```text
rule_id
created_by
approved_by
created_at
status
version
```

# 三、标签系统接口规范（API Contract）

本规范用于统一标签系统接口设计，确保系统间通信一致、稳定、可扩展。

---

## 3.1 API 设计基本原则

```text
统一路径
统一返回结构
统一错误码
支持扩展
```

---

## 3.2 基础路径规范

```text
/api/tag
/api/user/tag
/api/audience
```

---

## 3.3 核心接口列表（必须）

| 接口 | 方法 | 说明 |
|------|------|------|
创建标签 | POST | 创建标签定义 |
更新标签 | PUT | 修改标签 |
删除标签 | DELETE | 删除标签 |
获取标签列表 | GET | 查询标签 |
为对象添加标签 | POST | 添加标签 |
获取对象标签 | GET | 查询标签 |
删除对象标签 | DELETE | 删除标签 |

---

## 3.4 创建标签接口（示例）

### 请求

```http
POST /api/tag
```

```json
{
  "tag_code": "HEALTH_OBESE_HIGH_RISK",
  "tag_name": "高危肥胖",
  "tag_type": "风险标签",
  "expire_days": 365
}
```

---

### 返回

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "tag_id": 1001
  }
}
```

---

## 3.5 为对象添加标签接口（示例）

### 请求

```http
POST /api/user/tag
```

```json
{
  "user_id": 1001,
  "tag_code": "HEALTH_OBESE_HIGH_RISK",
  "source": "rule_engine"
}
```

---

### 返回

```json
{
  "code": 0,
  "message": "success"
}
```

---

## 3.6 获取对象标签接口（示例）

### 请求

```http
GET /api/user/tag?user_id=1001
```

---

### 返回

```json
{
  "code": 0,
  "data": [
    {
      "tag_code": "HEALTH_OBESE_HIGH_RISK",
      "tag_name": "高危肥胖",
      "expire_at": "2026-12-31"
    }
  ]
}
```

---

## 3.7 标准返回结构（统一）

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

---

## 3.8 标准错误码（推荐）

| code | 说明 |
|------|------|
0 | 成功 |
400 | 参数错误 |
401 | 未授权 |
404 | 未找到 |
500 | 系统错误 |
