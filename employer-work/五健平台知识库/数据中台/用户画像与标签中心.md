# 儿童青少年“五健”管理平台

## 标签管理系统（用户画像与标签中心）设计方案

---

一、建设背景（标签管理的必要性）
在儿童青少年“五健”（体重、视力、心理、骨骼、口腔）健康管理体系中，平台逐步从**数据采集系统**向**健康管理与运营系统**演进。仅依赖基础信息管理与筛查结果存储，已无法满足精细化健康干预、精准随访和持续运营服务的需求。
因此，必须建立统一的**用户画像与标签管理体系**，实现：
- 对儿童健康状态进行结构化识别
- 对重点人群进行自动识别与分层管理
- 对家长进行精准触达与健康干预
- 对学校与区域进行健康风险监测
- 支撑“五健”长期健康运营与随访管理

**标签体系本质上是：将数据转化为可运营资产的核心机制**

---
二、建设目标（标签管理系统的定位）
建设统一的**用户画像与标签中心（User Profile & Tag Center）**，实现以下目标：
 2.1 数据层目标
- 建立标准化标签字典体系
- 实现标签数据结构统一管理
- 支持标签生命周期管理
- 建立标签数据资产库

2.2 业务层目标
- 自动识别健康风险人群
- 自动识别重点随访对象
- 支持健康干预策略制定
- 支持学校健康管理决策

2.3 运营层目标
- 支持精准健康教育推送
- 支持复诊与随访提醒
- 支持家长健康干预
- 支持重点人群持续运营

2.4 技术层目标
- 支持规则引擎自动打标签
- 支持批量与实时标签生成
- 支持标签组合查询（人群圈选）
- 支持标签与AI模型集成

---
 三、标签管理系统总体架构

```text
标签管理系统（User Profile & Tag Center）
│
├── 1. 标签体系管理（Tag Dictionary）
├── 2. 标签生成引擎（Tag Engine）
├── 3. 用户标签管理（User Tagging）
├── 4. 人群圈选（Audience Segmentation）
├── 5. 标签应用中心（Tag Application）
├── 6. 标签治理中心（Tag Governance）
├── 7. 标签数据资产库（Tag Data Warehouse）
```

# 标签管理系统功能模块设计（技术方案级）

---

# 1 标签体系管理模块（Tag Dictionary）

## 功能说明

负责统一管理所有标签的定义，是标签系统的基础模块。

## 功能列表

### 1）标签分类管理

支持建立多级标签体系：

```text
基础标签
健康标签
风险标签
行为标签
运营标签
设备标签
AI预测标签
```

### 2）标签属性管理

每个标签必须具备标准属性：

|字段|说明|
|---|---|
|标签编码|唯一标识|
|标签名称|业务名称|
|标签类型|基础/健康/行为/风险等|
|标签值类型|枚举/数值/布尔|
|标签来源|规则/人工/模型|
|标签有效期|天/月/永久|
|标签状态|启用/停用|
|创建时间|系统生成|

### 3）标签生命周期管理

支持：

```text
创建
启用
停用
过期
删除
```

# 2 标签生成引擎（Tag Engine）

## 功能说明

负责根据规则或模型自动生成标签，是标签系统的核心能力。

## 支持生成方式

### 1）规则自动生成标签

示例：

```text
IF BMI > 95th percentile
THEN 标签 = 高危肥胖
```

```text
IF 裸眼视力 < 4.6
THEN 标签 = 视力异常
```

### 2）批量标签生成

例如：

```text
导入筛查数据后
自动生成健康标签
```

### 3）实时标签生成

例如：

```text
家长完成随访
自动生成：已完成随访
```

### 4）AI预测标签（高级阶段）

例如：

```text
近视风险预测
肥胖风险预测
心理异常风险预测
```

# 六、标签系统数据库表结构设计（ER级）

---

# 6.1 标签定义表（tag_definition）

```sql
CREATE TABLE tag_definition (

id BIGINT PRIMARY KEY,

tag_code VARCHAR(50) UNIQUE,

tag_name VARCHAR(100),

tag_type VARCHAR(50),

tag_value_type VARCHAR(50),

tag_source VARCHAR(50),

tag_expire_days INT,

tag_status VARCHAR(20),

created_at DATETIME,
updated_at DATETIME

);
```

---

# 6.2 用户标签关系表（user_tag_relation）

```sql
CREATE TABLE user_tag_relation (

id BIGINT PRIMARY KEY,

user_id BIGINT,
tag_id BIGINT,

tag_value VARCHAR(100),

source VARCHAR(50),

created_at DATETIME,
expire_at DATETIME

);
```

---

# 6.3 标签规则表（tag_rule）

```sql
CREATE TABLE tag_rule (

id BIGINT PRIMARY KEY,

rule_name VARCHAR(100),

tag_id BIGINT,

rule_expression TEXT,

rule_type VARCHAR(50),

status VARCHAR(20),

created_at DATETIME

);
```

---

# 6.4 人群定义表（audience_definition）

```sql
CREATE TABLE audience_definition (

id BIGINT PRIMARY KEY,

audience_name VARCHAR(100),

condition_expression TEXT,

created_at DATETIME

);
```

# 九、可直接用于技术方案的标准描述（建议直接使用）

本系统建设统一的用户画像与标签管理中心，基于筛查数据、随访数据、行为数据及健康风险评估模型，自动生成多维度标签，实现对儿童青少年健康状态、行为特征及风险等级的精准识别，并支撑健康干预、随访管理及精准运营服务，形成以数据驱动的健康管理与服务闭环。

建议按以下顺序推进：

1）先建设标签字典

2）再建设标签规则

3）再建设用户标签表

4）再建设人群圈选模块

5）最后建设运营推送模块

这是行业最稳定的实施路径。
