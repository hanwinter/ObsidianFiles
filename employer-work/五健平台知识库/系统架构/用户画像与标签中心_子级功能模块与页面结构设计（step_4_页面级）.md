# 一、本阶段目标（必须明确）

本阶段进入产品级设计，但仍然不是 UI 设计，而是：

```text
定义系统菜单与页面结构（Page Architecture）
```

目标是确保：

- 页面结构稳定
- 页面职责清晰
- 页面与数据实体一一对应
- 页面数量可控

# 三、页面设计原则（必须遵守）

```text
每个页面只解决一个核心问题
每个页面只操作一个主实体
页面名称必须反映职责
页面数量控制在可维护范围
```

# 五、页面复杂度控制规则（非常重要）

```text
一个模块页面数量 ≤ 6
一个页面字段数量 ≤ 30
一个页面操作按钮 ≤ 8
```

这是长期系统稳定运行的经验阈值。
