
# 用户画像与标签中心设计的关键核心记忆（用于新窗口快速恢复上下文）

---

# 一、系统定位（必须统一）

```text
用户画像与标签中心 = 平台级能力中心
```

不是：

```text
标签模块
```

---

# 二、系统服务对象（多主体）

```text
儿童
家长
学校
医生
机构
```

---

# 三、ID 体系规则（关键）

```text
不同主体使用不同业务 ID
```

例如：

```text
student_id
guardian_id
school_id
doctor_id
org_id
```

但系统内部统一使用：

```text
entity_id
```

---

# 四、系统必须支持的 6 个核心能力（不可删除）

```text
多主体支持
标签自动生成
标签自动失效
标签历史记录
跨主体触发
人群识别
```

---

# 五、系统核心实体（必须牢记）

```text
Entity
Tag
EntityTag
TagRule
Audience
EntityRelation
```

这是系统的根模型。

---

# 六、标签系统的 5 个设计原则（固定记忆）

```text
标签是属性，不是对象
用户是主体，标签是附属
标签必须带时间
标签必须记录来源
标签必须可失效
```

---

# 七、系统复杂度等级（当前）

```text
Level 4：运营级标签系统
```

特征：

```text
自动化
生命周期
多主体
规则驱动
审计能力
```

---

# 八、系统边界（必须明确）

---

## 本系统负责

```text
标签管理
标签规则
标签生命周期
用户画像
人群识别
标签查询
```

---

## 本系统不负责

```text
业务流程
业务数据录入
消息发送
报表展示
```

---

# 九、系统对接方式（固定模式）

```text
查标签：API
改标签：事件
做统计：数据仓库
```

---

# 十、系统设计的一句话总结（建议直接复制到新对话窗口）

```text
这是一个多主体、多ID体系、支持自动化与生命周期的用户画像与标签中心，
其核心实体包括 Entity、Tag、EntityTag、TagRule、Audience、EntityRelation，
系统以规则驱动标签生成，以事件驱动标签变化，并通过 API 向业务系统提供标签服务。
```
