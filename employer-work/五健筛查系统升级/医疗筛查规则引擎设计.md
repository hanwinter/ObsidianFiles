# 医疗筛查规则引擎设计 — 本次对话核心知识沉淀

---

# 一、整体目标

本次对话核心是在构建：

# “医疗筛查规则引擎”

用于：

```text
体检原始数据
→ 异常指标
→ 疾病规则判断
→ 阳性结果
→ 标签
→ 画像
```

适用于：

- 五健筛查
    
- 学生健康管理
    
- AI健康报告
    
- 风险画像
    
- 随访干预
    

---

# 二、核心架构演进

---

# 第一阶段（最初）

用户最初设计：

```text
体检项目
→ 疾病
```

问题：

- 疾病规则耦合在体检项目中
    
- 无法跨项目复用
    
- 难扩展
    

---

# 第二阶段（规则分层）

逐渐形成：

```text
体检子项结果
→ 异常指标
→ 疾病规则
→ 疾病结果
```

核心思想：

# 疾病不直接依赖体检项目

而是依赖“异常指标”。

---

# 第三阶段（规则引擎化）

最终形成：

```text
疾病
→ 条件组（OR）
→ 条件（AND）
```

这是最终收敛结构。

---

# 三、最终规则模型（核心）

---

# 1. Disease（疾病）

例如：

- 超重
    
- 肥胖
    
- 屈光不正
    
- 龋齿
    
- 脊柱侧弯
    

疾病是最终输出结果。

---

# 2. Condition Group（条件组）

多个条件组之间：

# OR

表示：

```text
满足任意一种判定路径
即可命中疾病
```

例如：

---

## 超重

### 条件组1

```text
年龄<=72月
AND
BMI>=P85
```

---

### 条件组2

```text
7岁<年龄<18岁
AND
BMI>=P85
```

条件组之间：

```text
OR
```

---

# 3. Condition（条件）

条件组内部：

# 默认 AND

表示：

```text
所有条件必须同时成立
```

---

# 四、条件分类（重要）

本次对话中最关键的架构升级之一：

# 区分：

---

# 1. Scope（规则适用范围）

不是疾病判断条件。

而是：

```text
规则适用于谁
```

例如：

|字段|示例|
|---|---|
|年龄|7~18岁|
|性别|男|
|学段|小学|

特点：

# Scope 永远是 AND

因为：

```text
规则必须同时满足适用范围
才会执行
```

---

# 2. Condition（阳性判断条件）

真正参与疾病判定。

例如：

|条件|
|---|
|BMI >= P85|
|裸眼视力 < 5.0|
|SPH <= -0.50|

---

# 五、规则执行流程（最终版）

---

# Step1：筛选适用规则（Scope）

根据：

- 年龄
    
- 性别
    
- 学段
    

过滤规则。

---

# Step2：执行阳性条件（Condition）

例如：

```text
BMI >= P85
```

---

# Step3：命中条件组

条件组内部：

```text
AND
```

---

# Step4：命中疾病

条件组之间：

```text
OR
```

任意组成立即可。

---

# Step5：生成建议结果

例如：

```text
建议结果：超重
```

---

# Step6：医生确认

医疗系统必须：

```text
医生最终确认
```

不能直接自动诊断。

---

# 六、规则逻辑设计原则（核心）

---

# 1. 不穷举规则组合

错误思路：

```text
组合1
组合2
组合3
...
```

原因：

# 医疗组合无限增长。

---

# 正确方式：

# 条件表达式

例如：

```text
(A AND B)
OR
(C)
```

---

# 2. 有限DSL

不做：

# 通用Drools规则平台

而是：

# 有限医疗DSL

原因：

- 医生不会编程
    
- 配置复杂度会爆炸
    

---

# 3. 优先“规则组”

而不是：

# 无限表达式树

推荐：

```text
组内 AND
组间 OR
```

即可覆盖大多数医疗场景。

# 十、参考标准（核心）

本次对话非常重要的内容：

# 医疗规则很多不是固定值

而是：

# 动态医学标准

---

# 推荐支持的参考标准类型

---

# 1. FIXED_VALUE

固定值。

例如：

```text
BMI > 28
```

---

# 2. REFERENCE_RANGE

正常参考范围。

例如：

```text
视力 < 正常下限
```

---

# 3. PERCENTILE_STANDARD

百分位标准。

例如：

```text
BMI >= P85
```

依赖：

- 年龄
    
- 性别
    

---

# 4. AGE_SEGMENT_STANDARD

年龄分段标准。

例如：

```text
不同年龄视力标准
```

---

# 5. GROWTH_CURVE_STANDARD

生长曲线标准。

例如：

```text
BMI Z-Score
```

---

# 十四、最终架构结论（本次对话最核心）

---

# 疾病规则：

# 不属于体检项目

而是：

# 独立规则中心

---

# 疾病规则：

# 基于异常指标

不是原始项目。

---

# 规则结构：

# 条件组 OR + 条件 AND

即可满足绝大多数医疗场景。

---

# 年龄/性别：

# 属于 Scope（规则适用范围）

不是疾病条件本身。

---

# 动态参考标准：

# 是医疗规则系统的核心复杂度来源。

---

# 当前阶段：

# 不要做通用规则平台

优先：

# 有限医疗DSL + 固定规则结构。
